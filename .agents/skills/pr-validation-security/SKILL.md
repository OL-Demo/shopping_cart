---
name: pr-validation-security
description: This should be used to validate the new changes made in a PR are secure. Nothing has been committed that shouldn't be committed.
---

Ensure that the changes made on the specified PR are secure. Ensure no tokens committed, no internal files are committed, and that no files that shouldn't have been committed have been committed.

Also do a scan for some common security issues that people make.
